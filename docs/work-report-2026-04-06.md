## Безопасность и платёжная система

Проведён полный аудит безопасности и устранены критические уязвимости:

- Платёжные вебхуки YooKassa теперь проверяются через прямой запрос к API, а не просто по IP-адресу — исключена возможность подделки оплаты
- Исправлена проблема с одновременными заказами, при которой один товар мог быть продан дважды — теперь списание со склада происходит атомарно на уровне базы данных
- При сбое платёжной системы заказ автоматически откатывается: товар возвращается на склад, бонусы — на баланс клиента, корзина сохраняется
- Цена доставки теперь всегда рассчитывается сервером — исключена возможность подмены цены со стороны клиента
- Добавлен автоматический возврат средств через YooKassa при отмене оплаченного заказа
- Все секреты и ключи вынесены из кода, приложение не запустится без корректно заполненных переменных окружения
- Добавлены заголовки безопасности: защита от кликджекинга, принудительный HTTPS, политика контента

## Модуль доставки

СДЭК полностью доведён до рабочего состояния:

- Поиск городов с кешированием токена авторизации и защитой от таймаутов
- Тарифы определяются автоматически по ответу API (не захардкожены) — работает с любым тарифом
- Карта пунктов выдачи на Яндекс.Картах с маркерами, выбор точки кликом
- Вебхук СДЭК логирует все изменения статусов в историю заказа
- Смена учётных данных СДЭК в админке мгновенно сбрасывает кеш токена

Почта России также подключена:

- Расчёт тарифов с автоматическим определением почтового индекса по названию города
- Поиск почтовых отделений по городу
- Логирование ошибок для диагностики проблем

## Инфраструктура и деплой

- Приложение работает в Docker на сервере Beget (порт 3004)
- База данных закрыта от внешнего доступа — доступна только внутри Docker-сети
- Все переменные окружения заполнены: OAuth (Google, Yandex, VK), YooKassa, SMTP, вебхук-секреты СДЭК
- Миграции базы данных применены
- 6 API-эндпоинтов обёрнуты в обработку ошибок для стабильной работы
