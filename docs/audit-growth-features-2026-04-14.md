# Аудит фич роста главной страницы

**Дата:** 2026-04-14
**Scope:** проверка всего что добавлено в коммите `cfe0d31` (SocialProofStrip, FreshnessBadge, CategoryTiles, StickyMobileCTA, FeaturedBlog, TasteQuiz + API).

Оценка по: баги, отсутствие фронтенда/кнопок, несоответствие дизайн-системе.

---

## 🔴 Критические / потенциальные баги

### 1. SocialProofStrip — выдуманные числа (легальный риск)
`components/home/SocialProofStrip.tsx` хардкодит `rating=4.9`, `reviewsCount=2480`, `ordersShipped=12400`.
**Риск:** ЗоЗПП РФ + закон о рекламе — недостоверные сведения о товаре/услуге. При жалобе можно получить предписание ФАС или Роспотребнадзора.
**Fix:** либо (a) убрать точные числа и заменить на качественные формулировки («Тысячи довольных клиентов», «Отличные отзывы»), либо (b) подтягивать реальные значения из БД (`reviews count from Review table`, `orders count from Order table` status='delivered').

### 2. StickyMobileCTA перекрывает контент
Фиксированная плашка ~72px высотой. На страницах где она активна (главная), футер/последние секции частично закрываются при скролле.
**Fix:** добавить `pb-20 md:pb-0` на `<main>` в `app/page.tsx` — либо CSS-переменная-компенсатор в body через эффект.

### 3. Quiz match %: `Math.min(99, …)` обрезает идеальное совпадение
`QuizResult.tsx:52` — при score=100 показывает «Совпадение 99%». Противоречит «идеальный подбор».
**Fix:** `Math.min(100, score)`.

### 4. Freshness timezone
`lib/freshness.ts` использует `new Date()` на сервере. Docker-контейнер по умолчанию UTC. В 23:30 Мск (20:30 UTC) дата ещё «вчерашняя UTC» → плашка может показать устаревшее «вчера» пользователю в Мск.
**Fix:** либо зафиксировать `Europe/Moscow` при расчёте (`toLocaleString("ru-RU", {timeZone: "Europe/Moscow"})`), либо переключить расчёт на клиентскую сторону с TZ пользователя.

### 5. Quiz scoring: нет гарантированного фолбэка при score < 40
В плане было «если меньше 3 совпадений с score>=40 — добираем по общему рейтингу». Сейчас API просто сортирует и берёт топ-3 без порога. Тест: если каталог пустой или товары не подходят, вернётся 3 случайных — некорректный UX. Сейчас это ОК для небольшого каталога, но с ростом превратится в проблему.
**Fix:** в `scoring.ts` добавить порог. Если после фильтра < 3 — добирать по `averageRating DESC`.

### 6. Quiz API: нет валидации входа, нет rate-limit
`app/api/quiz/match/route.ts` принимает любой body. Загрузка ВСЕХ активных товаров с reviews/variants — дорогой запрос. Можно спамить эндпоинт.
**Fix:** zod-валидация answers (или ручная), `cache` 60 сек на scorable-данные продуктов, опционально — базовый IP rate-limit.

### 7. Intersection Observer: `heroVisible` по умолчанию `true` — если Hero ещё не в DOM
`StickyMobileCTA.tsx:15` — если IO не успевает подписаться до рендера страницы, `heroVisible` остаётся `true` и CTA не появляется. На практике работает, но хрупко.
**Fix:** проверять `hero?.getBoundingClientRect()` при маунте + делать первоначальную оценку.

## 🟡 Отсутствие кнопок / секций

### 8. SocialProof: рейтинг не кликабелен
Плашка «★ 4.9 · 2 480 отзывов» выглядит как социальное доказательство, но не ведёт к отзывам. Пользователь кликает — ничего не происходит.
**Fix:** обернуть первый item в `<Link href="#testimonials">`.

### 9. CategoryTiles: нет catch-all плитки «Все сорта»
Если пользователь не варит ни одним из 6 перечисленных способов — он застревает. Нужна 7-я плитка «Не знаю / показать все» → `/catalog`.

### 10. Quiz intro: нет skip-ссылки
Пользователь который точно знает что хочет, вынужден проходить 5 шагов. Нужна ссылка «У меня уже есть фаворит → в каталог» под CTA-кнопкой.

### 11. QuizResult: нет «Поделиться подборкой»
Результат квиза — сильный UX-момент. Share-кнопка (WhatsApp/Telegram/copy URL) расширяет охват.

### 12. FreshnessBadge: нет тултипа «что это значит»
«Последняя обжарка: сегодня» — звучит странно пока не поймёшь «обжариваем под заказ». Нужен hint/icon `?` с краткой пояснительной подписью при ховере/тапе.

### 13. StickyMobileCTA: нет dismiss/close
Если пользователь хочет пролистать без отвлекающего баннера — не может. Добавить маленькую × справа → localStorage «закрыто на эту сессию».

### 14. StickyMobileCTA: «От 550₽» — хардкод
Если минимальная цена изменится, не обновится. Нужно либо (a) убрать цифру, либо (b) подтягивать через API `/api/products/min-price` (SSR при моунте клиента).

### 15. FeaturedBlog: 2 дубликата «Все статьи»
Мобильный + десктопный вариант `ArrowRight` ссылки. Можно объединить в одну responsive-версию.

## 🟠 Дизайн-система: несоответствия

### 16. CategoryTiles: неконсистентные подписи
Сейчас: «**Для** эспрессо», «**Для** фильтра», «**Для** турки», «Френч-пресс» (без «Для»), «Мока» (без «Для»), «Аэропресс» (без «Для»).
**Fix:** унифицировать — либо все с «Для», либо все без. Оптимально без префикса для краткости: «Эспрессо / Фильтр / Турка / Френч-пресс / Мока / Аэропресс».

### 17. CategoryTiles hover: тёмная заливка `bg-foreground` агрессивна
Весь остальной сайт в тёплой палитре. Ховер делает тайл чёрным — визуальный шок. Соответствие: ховер через `bg-primary/5` или `border-primary`, как в `QuizStep`.

### 18. SocialProof: янтарная звезда выбивается из палитры
Остальные иконки `text-primary` (коричневый). Звезда — amber-400 (жёлтый). Рейтинговая звезда конвенциональна жёлтая, но **контрастирует** с коричневой темой. Альтернатива: `fill-primary text-primary` (кофейная звезда) или оставить amber для привычности.

### 19. Quiz section padding: `lg:py-24` vs остальные `lg:py-20`
Секционный ритм ломается на 4px. Единственная секция с `py-24` = TasteQuiz. Унифицировать на `lg:py-20`.

### 20. QuizResult badge «Совпадение X%» перекрывается с product card badge
`ProductCard` имеет свой badge (`top-2 left-2`) для «Хит продаж» / «Распродажа». QuizResult клеит поверх свой «Совпадение 87%» (`-top-2 left-2`). **Badges конфликтуют — 2 плашки в одной точке.**
**Fix:** либо (a) в QuizResult временно скрывать product badge, либо (b) двигать match-badge в `top-right` (где favorite button — тоже конфликт), либо (c) внутрь карточки полосой над ценой.
Оптимально: заменить абсолютный badge на inline-плашку внутри карточки после имени товара.

### 21. Quiz h3 должен быть h2
`QuizStep` рендерит `<h3>` вопроса, но секция `TasteQuiz` не имеет h2 внутри фазы `questions` (intro имеет). Семантика ломается когда intro пропадает.
**Fix:** `<h2>` в обоих phases — или добавить visually-hidden h2 на уровне секции.

### 22. Quiz intro / result / questions — разная рамка-контейнер
Intro — без контейнера. Questions — `bg-secondary/20 rounded-2xl p-5-10`. Loading — такой же контейнер. Result — без контейнера.
**Fix:** либо все фазы в одинаковой раме, либо интро/result на чистом фоне последовательно. Сейчас визуальная нелогичность.

### 23. FreshnessBadge: на мобиле разделитель `·` скрыт, элементы просто в колонке
`sm:inline text-primary/30` — ок, но gap между двумя строками на мобиле `gap-2` = 8px — тесновато. Можно `gap-3` на mobile.

## 🟢 UX / полиш

### 24. Quiz: нет автоскролла к началу шага
После клика «Далее» новый шаг появляется на том же scroll-position. На мобиле заголовок шага может быть ниже экрана.
**Fix:** `scrollIntoView({ behavior: "smooth", block: "start" })` на контейнер при смене step.

### 25. Quiz: keyboard arrow-keys не работают в radiogroup
`role="radiogroup"` + `role="radio"` ожидают стрелочную навигацию. Сейчас только Tab/Space.
**Fix:** onKeyDown handler, `ArrowDown/ArrowUp/ArrowRight/ArrowLeft` для перехода между опциями.

### 26. Quiz: отсутствие аналитики
В плане были комменты `// TODO: analytics: <event>`. Забыл вставить хуки `ym(...)` для `quiz_start`, `quiz_step`, `quiz_complete`, `quiz_cta_click`.

### 27. StickyMobileCTA: безальтернативная метка
«Выбрать кофе» — хорошо, но после квиза и категорий на той же странице это уже 5-я CTA-кнопка в каталог. Дублирование.
**Fix:** показывать CTA только если пользователь не кликал на Hero/Category/Quiz CTA (сложно), или просто принять дублирование.

### 28. QuizResult: нет индикации «Идеальный / хороший / подойдёт» уровней
% совпадения — абстрактное число. Добавить словесный уровень: `85%+ → «Идеально вам»`, `70-85% → «Отличный выбор»`, `<70% → «Подойдёт»`. Повышает доверие.

### 29. CategoryTiles: при клике на метод с 0 товарами — пустой каталог
Если в БД нет товаров с `brewingMethods.has("moka")` — клик на плитку «Мока» ведёт на пустой каталог с EmptyState. Нужно (a) скрывать плитки без товаров (запрос count), либо (b) показывать count на плитке «Мока (3)».

### 30. SocialProof на десктопе: 4 элемента через `lg:justify-between`
На 1280px+ элементы слишком расползаются. Лучше `lg:gap-12 justify-center`.

### 31. Секция Quiz имеет слабый визуальный отделитель
Если Products и Quiz идут подряд, обе с `bg-white` / `bg-white` — границы между ними размываются. В плане был chained ритм через чередование фонов.
**Fix:** Quiz section → `bg-secondary/30` или добавить visual separator.
